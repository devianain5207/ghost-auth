# Code of Conduct

## Ghost Auth — Community Standards

Ghost Auth is maintained by **Kestrel AS** and built by contributors from around the world. This Code of Conduct applies to all project spaces — GitHub issues, pull requests, discussions, and any other communication channels associated with Ghost Auth.

## Our Standards

We want Ghost Auth's community to be straightforward and productive. That means:

### Expected Behavior

- Be respectful and constructive in all interactions
- Give and receive feedback gracefully — code review is about the code, not the person
- Assume good intent, especially from new contributors
- Stay on topic — keep discussions relevant to the project
- Credit others' work and ideas
- Follow the project's [Contributing Guidelines](CONTRIBUTING.md) and [Security Policy](SECURITY.md)

### Unacceptable Behavior

- Harassment, intimidation, or discrimination of any kind
- Personal attacks, insults, or derogatory language
- Publishing others' private information without consent
- Sustained disruption of discussions or review processes
- Spam, self-promotion, or off-topic solicitation
- Sexualized language or imagery in any project space
- Any conduct that would be considered inappropriate in a professional setting

## Scope

This Code of Conduct applies to all interactions within the Ghost Auth project. It also applies when an individual is representing the project in public spaces — for example, using an official project account or acting as an appointed representative at an event.

## Enforcement

Instances of unacceptable behavior can be reported by contacting the project maintainers at **ghostauth@kestrel.no**.

All reports will be reviewed and handled confidentially. The maintainers are obligated to maintain the privacy of the reporter.

### Enforcement Actions

Maintainers may take any action they deem appropriate, including:

1. **Correction** — A private message clarifying the issue and expected behavior.
2. **Warning** — A formal warning with consequences for continued behavior.
3. **Temporary ban** — Temporary exclusion from project spaces.
4. **Permanent ban** — Permanent removal from all project spaces.

The severity of the response will match the severity and context of the violation.

## Maintainer Responsibilities

Project maintainers are responsible for clarifying and enforcing these standards. They have the right and responsibility to remove, edit, or reject comments, commits, code, issues, and other contributions that do not align with this Code of Conduct.
